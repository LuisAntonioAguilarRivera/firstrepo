Instrucciones Git

A) Clonar un repo con la sentencia git clone, normalmente nos dejará en la rama proincipal una vez que lo clone en local

B) Crear una rama develop donde trabajaremos en forma local con la sentencia git branch develop

C) Cambiarnos de rama a develop con git checkout develop

D) Validar el estatus del repositorio con el comando git status

E) Realizar los cambios en los fiucheros donde se necesite

E) Agregar archivos a Staging con el comando git add nombre del archivo

F) Crear el commit con esta instrucción: git commit -m "Your message about the commit"

G) Cambiar a rama main con git checkout main

I) Unir las ramas con un git merge

H) Para realizar un push se debe agregar la siguiente sentencia:

git push

Solicitará autenticación con el usuario y con un token que debe ser generado en la página de GitHub:

Settings->Developer settings->Personal access tokens

Ahí generar el token y darle permisos para modificar repositorios.

Esto es una prueba
